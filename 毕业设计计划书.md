# 基于多因子融合与LLM交互的量化信号分析系统——本科毕业设计行动计划书

## 一、项目概述

### 1.1 项目定位
本项目定位为**量化信号分析与风险预警系统**，而非传统意义上的自动交易系统。系统核心功能是对市场数据进行多维度信号提取与风险研判，并通过大语言模型（LLM）驱动的智能交互界面，向用户提供可理解、可对话的市场状态解读。

### 1.2 核心创新点
1. **信号处理与金融因子融合**：将电子信息工程领域的时域/频域信号分析方法引入量化分析框架，作为传统多因子模型的补充，用于市场风险预警。
2. **LLM前置化交互设计**：将大语言模型置于前端，作为“系统解说员”角色，负责自然语言交互与分析结果解读，不参与后台量化计算与投资决策，规避LLM在金融领域的幻觉与实时性问题。

### 1.3 政策契合点
- 响应“数字金融”发展号召，为投资者提供数字化决策参考工具。
- 符合“人工智能+金融”的规范应用导向，明确划分AI在金融领域的职责边界。
- 系统定位为**风险监测与预警**，服务于资本市场内在稳定性建设，与当前监管引导方向一致。

### 1.4 专业交叉价值
- **金融知识**：多因子模型（估值、动量、质量因子等）、Alpha/Beta收益分解。
- **电子信息工程**：信号时域能量分析、频域FFT/小波变换、自相关/互相关分析。
- **计算机科学**：系统架构设计、实时数据处理、LLM API集成与Prompt Engineering。


## 二、系统架构设计（四层结构）

### 2.1 数据接入层
- **功能**：获取与清洗原始金融数据。
- **数据源**：
  - A股行情数据（基于日K分析）
  - 财务数据（季报/年报中的PE、PB、ROE等）
  - 舆情数据（可选，作为辅助因子来源）
- **技术选型**：Python（Requests/Akshare/Tushare）、WebSocket实时行情接入。

### 2.2 信号分析层（核心层）
本层分为两个并行的分析模块，其结果将共同输入融合层：

#### 模块A：传统金融多因子模块
- **估值因子**：PE、PB、PS等，判断标的相对估值水平。
- **动量因子**：过去N日收益率、乖离率等，捕捉价格趋势。
- **质量因子**：ROE、资产负债率等，衡量企业基本面健康度。
- **贝塔因子**：标的相对于大盘的波动敏感度。

#### 模块B：信号处理因子模块（独门创新）
本模块将电子信息工程的信号分析方法引入金融场景，因子分为两类用途：

**温度计类（市场状态描述，无预警领先性）：**
- **时域能量分析**：
  - 将价格序列视为时域信号，计算滑动窗口内的短期能量与长期能量之比。
  - 展示逻辑：比值越高，表示近期波动强度相对常态越剧烈，用于展示"市场当前处于高波动状态"。
- **频域分析（FFT）**：
  - 对收益率序列进行频谱分析，观察能量在低频段（长期趋势）与高频段（市场噪声）的分布比例。
  - 展示逻辑：高频占比高，表示市场噪声成分多、方向感弱，用于展示"当前市场情绪紊乱程度"。

**预警类（中长期结构判断，有统计上的领先性）：**
- **自相关分析**：
  - 计算不同滞后阶数下收益率序列的自相关系数，衡量趋势动能的持续性。
  - 预警逻辑：自相关系数由正转负，表示长期趋势动能衰竭、结构开始瓦解，是中长期重大回撤的领先信号（经沪深300全历史验证，命中率提升1.45x，优于RSI/布林带基准）。

### 2.3 多因子融合与预警层
- **功能**：将上述因子的输出进行综合评分，输出最终风险预警等级。
- **预警等级设计**：无 / 关注 / 警惕 / 危险（四级）。
- **融合方式**：规则引擎（可解释性强），以自相关因子为预警主力，传统多因子作补充。
- **输出内容**：
  - 综合预警等级（由预警类因子驱动）。
  - 市场状态快照（由温度计类因子提供：当前高频占比、能量比读数及变化趋势）。
  - 触发预警的关键因子及其贡献度说明。

### 2.4 LLM智能交互层（前端Agent）
- **角色定位**：系统解说员，只读数据，不预测未来。
- **工作流程**：
  1. 用户通过聊天界面输入自然语言问题。
  2. LLM Agent对问题进行意图识别，提取关键实体（如时间、标的、指标类型）。
  3. Agent调用后端API获取对应的信号分析数据。
  4. Agent将结构化数据组织为通俗易懂的自然语言回复，向用户“讲解”系统分析结论。
- **核心原则**：所有结论性内容均来自后端量化计算，LLM只负责表达层的翻译与呈现，不自主生成投资建议。
- **典型交互场景**：
  - 用户："当前市场整体风险水平如何？"
  - Agent调取预警等级与温度计数据后回复："根据系统分析，当前市场综合风险为'关注'级别。自相关因子当前处于负值区间，显示趋势动能有所减弱。从市场状态来看，沪深300近250日高频能量占比为52%，能量比为1.3，波动略高于常态水平。建议关注持仓结构的安全边际。"
  - 用户："这个预警是怎么触发的？"
  - Agent调取具体因子分解数据后回复："触发预警的核心因子来自自相关分析模块：沪深300收益率序列的滚动自相关系数近期由正转负，表明趋势动能出现衰竭迹象。历史统计显示，此类结构性转变往往领先于中长期重大回撤。温度计层同时显示当前市场噪声水平偏高，整体市场情绪偏向无序。"

## 四、验证方案（如何证明系统有效）

### 4.1 功能验证
- 展示系统能正常接入数据、计算因子、输出预警等级。
- 展示LLM Agent能正确理解用户意图并调用API返回结构化解读。
- 通过界面截图和操作录屏进行功能演示。

### 4.2 有效性验证（预警能力）
- 选取A股包含典型牛市顶部或系统性大跌的历史区间（如2015年股灾、2021年抱团顶）。
- 将系统预警信号标注在K线图上，观察预警是否出现在重大回撤之前。
- 统计预警的**命中率**（定义：预警发出后未来60交易日内最大回撤超过15%即为命中）与**基准对比**（自相关因子命中率1.45x，优于RSI/布林带）。

### 4.3 对比基准
- 与传统布林带或RSI等单指标阈值方法进行对比，展示本系统多因子+信号处理融合的预警时效性和准确率提升。


## 五、技术选型建议

| 层级 | 技术方案 |
|------|----------|
| 后端框架 | Python FastAPI（轻量、异步支持好） |
| 数据处理 | Pandas、NumPy、SciPy（FFT/小波/相关分析） |
| 数据获取 | Tushare Pro / AkShare / 券商量化API |
| 前端界面 | Vue3 + Element Plus（或直接使用Streamlit快速搭建） |

| 可视化 | Plotly / ECharts（用于展示频谱分析图和预警信号图） |

## 七、关键注意事项

1. **大模型的定位要始终坚持“前端解说员”**，在论文中反复强调LLM不参与量化决策，避免被质疑金融场景下LLM的可靠性与合规性问题。
2. **信号处理部分要有明确的金融映射逻辑**，不能为了用信号分析而用，每一种变换或指标都需要解释其在金融语境下的含义（例如：高频能量→市场情绪噪声程度）。
3. **预警验证要客观**，不追求100%准确，重点展示系统在某些典型场景下确有提前发现异常的能力，并诚实分析漏报和误报案例。
4. **数据获取需提前确认可用性**，建议优先选择有稳定免费接口的数据源（如Tushare积分权限、AkShare），避免后续因数据断供影响进度。
